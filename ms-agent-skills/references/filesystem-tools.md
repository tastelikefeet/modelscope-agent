# Filesystem Tools Capability

## Scope

This page documents the **capability-layer** file editing tools exposed in the
skill index:
- `replace_file_contents`
- `replace_file_lines`

Lower-level `file_system` primitives such as `write_file`, `read_file`, and
`list_files` belong to the base tool layer. They may be available in other
contexts such as delegated agents, but they are **not** additional capability
tools defined on this page.

Use `write_file` only when you truly want to create or overwrite an entire
file. For targeted edits, prefer the capability tools below.

## When to Use

Activate these tools when:
- Multiple agents may edit the same file concurrently
- You need precise, surgical text replacement without rewriting the whole file
- You want to insert content at the beginning or append at the end of a file
- You need to replace a specific occurrence of repeated text

## Tool: `replace_file_contents`

**Granularity:** Tool (atomic)
**Estimated Duration:** instant

The flagship editing tool. Finds an exact substring in a file and replaces
it with new content. Because it matches by **content** instead of line
numbers, it is **concurrent-safe** -- two agents can edit different parts
of the same file without conflicts.

### Parameters

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `path` | string | yes | -- | File path (relative to workspace or absolute) |
| `source` | string | yes | -- | Exact text to find (must match verbatim) |
| `target` | string | yes | -- | Replacement text |
| `occurrence` | integer | no | 1 | Which occurrence to replace (1-based). Use -1 for all. |

### Examples

**Replace a single function:**
```
replace_file_contents(
    path="src/utils.py",
    source="def old_function():\n    return 42",
    target="def new_function():\n    return 100",
    occurrence=1
)
```

**Replace all occurrences of an import:**
```
replace_file_contents(
    path="src/app.py",
    source="from old_module import helper",
    target="from new_module import helper",
    occurrence=-1
)
```

**Replace a specific (2nd) occurrence:**
```
replace_file_contents(
    path="config.yaml",
    source="enabled: true",
    target="enabled: false",
    occurrence=2
)
```

### Important Notes

- `source` must match the file content **exactly**, including:
  - Whitespace (spaces, tabs, newlines)
  - Punctuation style (e.g., Chinese `、` vs English `,`)
  - Unicode characters
- If the exact text is not found, the tool returns an error.
  Read the file first to get the precise content to match.
- When `occurrence` exceeds the total count, an error is returned.

### Why Use This Over the low-level `write_file` primitive?

The `write_file` name below refers to the lower-level `file_system` tool, not
to another capability listed on this page.

| Scenario | `write_file` | `replace_file_contents` |
|---|---|---|
| Rewrite entire file | Best choice | Overkill |
| Change one function in a 500-line file | Wasteful | Precise |
| Two agents editing different functions | **Conflict risk** | **Safe** |
| Rename a variable everywhere | Must rewrite all | Use `occurrence=-1` |

## Tool: `replace_file_lines`

**Granularity:** Tool (atomic)
**Estimated Duration:** instant

Replaces a range of lines by line number. Useful when you know the exact
line range and concurrency is not a concern.

### Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `path` | string | yes | File path |
| `content` | string | yes | New content |
| `start_line` | integer | yes | Start line (1-based). 0 = insert at beginning, -1 = append |
| `end_line` | integer | conditional | End line (1-based). Required when start_line > 0 |

### Special Modes

**Insert at beginning** (`start_line=0`):
```
replace_file_lines(
    path="src/main.py",
    content="#!/usr/bin/env python3\n# Generated by ms-agent\n",
    start_line=0
)
```

**Append at end** (`start_line=-1`):
```
replace_file_lines(
    path="CHANGELOG.md",
    content="\n## v1.2.0 - 2026-03-18\n- Added new feature\n",
    start_line=-1
)
```

**Replace lines 10-15:**
```
replace_file_lines(
    path="src/config.py",
    content="NEW_SETTING = True\nDEBUG = False\n",
    start_line=10,
    end_line=15
)
```

### Caveat

After a line-based replacement, subsequent line numbers may shift.
If performing multiple line-range edits on the same file, work from
**bottom to top** to avoid invalidating line numbers, or prefer
`replace_file_contents` which is immune to this issue.

## Choosing the Right Tool

```
Need to edit a file?
  │
  ├── Creating or overwriting the entire file?
  │   └── use low-level write_file (outside this capability page)
  │
  ├── Know the exact text to change?
  │   ├── Yes → replace_file_contents (concurrent-safe)
  │   └── No, but know the line numbers → replace_file_lines
  │
  └── Inserting at top or appending at bottom?
      └── replace_file_lines (start_line=0 or -1)
```
